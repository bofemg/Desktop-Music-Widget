✅ 项目优点：

快速上手：无需复杂的环境配置，通过底层句柄抓取和模拟按键，实现了对主流音乐软件的轻量级“遥控”。

交互灵动：独立实现了物理阻尼滚动算法（Spring Animation），让歌词切换具有丝滑的视觉过渡感，而非生硬的跳转。

多线程架构：将 UI 渲染、窗口监听、歌词爬取和封面下载分在不同线程，保证了桌面挂件在弱网环境下也不会卡死界面。

细节控：支持屏幕边缘自动吸附与隐藏，尽可能减少对日常学习、工作空间的遮挡。

⚠️ 现有不足（待改进）：

性能消耗：由于目前使用 UIA 接口实时监听进度条位置，在部分低配机器上可能会有一定的 CPU 占用，未来计划探索更轻量级的内存读取方案。

兼容性限制：目前主要针对酷狗音乐进行适配，对其他音乐平台的标题抓取逻辑还不够健壮。

代码整洁度：作为大二学生的探索项目，部分逻辑嵌套较深，缺乏完善的单元测试，代码结构仍有很大的优化空间。

🚀 后续优化计划
多平台适配：计划引入更通用的媒体控制接口（如 Windows Global Media Controls），实现对网易云、QQ音乐甚至网页版播放器的通用适配。

UI 皮肤系统：支持用户自定义背景透明度、金边颜色以及字体样式，打造更个性化的桌面挂件。

性能精简：优化轮询监听频率，引入更高效的进程间通信方式降低资源占用。
